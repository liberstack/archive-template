# JavaScript: Origens e Evolução

JavaScript foi criado em 1995 por **Brendan Eich**, engenheiro da Netscape Communications. Segundo relatos da própria empresa, a linguagem foi desenvolvida em apenas dez dias, com o objetivo de adicionar interatividade às páginas HTML exibidas no navegador Netscape Navigator.

## Nome e contexto histórico

A linguagem foi inicialmente chamada de **Mocha**, depois renomeada para **LiveScript** e, por fim, para **JavaScript** — um nome escolhido em parte por razões de marketing, aproveitando a popularidade da linguagem Java na época. Apesar do nome semelhante, JavaScript e Java são linguagens com sintaxes, propósitos e modelos de execução distintos.

## Padronização: ECMAScript

Para evitar fragmentação entre diferentes implementações de navegadores, a Netscape submeteu a linguagem à **ECMA International** em 1996. Isso resultou na criação do padrão **ECMAScript**, cuja primeira edição foi publicada em 1997. Desde então, JavaScript é a implementação mais conhecida da especificação ECMAScript.

## Guerra dos navegadores

No fim dos anos 1990, a disputa entre Netscape Navigator e Internet Explorer (guerra dos navegadores) resultou em implementações incompatíveis de JavaScript e do DOM (Document Object Model), dificultando o desenvolvimento de sites que funcionassem de forma consistente em diferentes navegadores.

## ES5 e a maturidade da linguagem

Depois de um período de estagnação, o padrão **ECMAScript 5** (ES5), publicado em 2009, trouxe melhorias importantes, como modo estrito (`strict mode`), métodos nativos para manipulação de arrays e suporte mais consistente a JSON.

## ES6 (ES2015): a virada moderna

Em 2015, o **ECMAScript 6**, também chamado de ES2015, introduziu mudanças estruturais significativas:

- Declarações `let` e `const`
- Arrow functions
- Classes
- Módulos (`import` / `export`)
- Template literals
- Promises nativas

A partir dessa versão, o TC39 (comitê responsável pela especificação ECMAScript) passou a lançar uma nova edição anual, identificada pelo ano (ES2016, ES2017, e assim por diante).

## Node.js e o JavaScript fora do navegador

Em 2009, **Ryan Dahl** criou o **Node.js**, um ambiente de execução baseado no motor V8 do Google Chrome, que permitiu rodar JavaScript no servidor, fora do contexto do navegador. Isso abriu caminho para o uso da linguagem em back-end, ferramentas de linha de comando e automação.

## JavaScript hoje

Atualmente, JavaScript é utilizado tanto no front-end quanto no back-end, em aplicações desktop (via Electron), aplicativos móveis (via frameworks híbridos) e ferramentas de build. A linguagem continua evoluindo por meio do processo aberto do TC39, com propostas públicas discutidas antes de serem incorporadas ao padrão.
